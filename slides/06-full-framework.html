models.py — Finding
@dataclass
class Finding:
    scenario_id: str
    title: str
    classification: Literal[
        "bug",             # file a bug
        "expected-failure", # known issue
        "flaky",            # timing issue
        "environment"       # infra problem
    ]
    severity: Literal["critical","high","medium","low"]
    description: str
    suggested_action: str
Classification maps to real QA workflow

bug → file a report  ·  expected-failure → skip/update test  ·  flaky → add wait  ·  environment → investigate infra. The Asserter does the triage.

prompt = (
    f"Scenario: {scenario.title}\n"

    #   With it, it has an explicit target to compare against.

response = client.messages.create(
    tools=[_CLASSIFY_TOOL],

    messages=[{ "role": "user", "content": prompt }],
    ...
)
expected_outcome is the Asserter's anchor

The Asserter compares actual behavior to expected_outcome. Without it, failure classification is guesswork. With it, the analysis is grounded.

reporter.py — write_markdown()
def write_markdown(report, path="report.md"):
    ts = datetime.now().strftime("%Y-%m-%d %H:%M UTC")
    lines = [
        f"# Test Report — {ts}",
        "| Total | Passed | Failed | Bugs |",
        f"| {report.total} | {report.passed} | ...",
    ]
    for r in report.results:
        icon = "✅" if r.passed else "❌"
        lines.append(f"### {icon} [{r.scenario_id}] {r.title}")
    if report.findings:
        for f in sorted_by_severity(report.findings):
            lines.append(f"### [{f.severity.upper()}] {f.title}")
            lines.append(f"- **Classification:** {f.classification}")
            lines.append(f"- **Next action:** {f.suggested_action}")

The Reporter is the only agent that calls no APIs. It's pure formatting logic. Change the output format without touching any other agent.

Module 06 · Key Takeaways

Six things
to carry forward.

1
Focused system prompts produce better reasoning
One responsibility per agent. Narrow scope means clearer thinking.
2
Data model = API contract
TestScenario, RunnerResult, Finding — define first, code second.
3
tool_choice appears twice
Architect and Asserter both use it. Schema validation at the API boundary, both times.
4
Priority-aware: fail fast on high
Smoke tests gate regression tests. Saves time and API cost on large suites.
5
Asserter runs only on failures
Spend AI compute where analysis matters — not on passing scenarios.
6
Reporter writes two files
report.md for humans, report.json for machines. Both become CI artifacts in Module 07.
