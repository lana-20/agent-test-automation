The Tradeoff

No getByRole —
so we build one.

Selenium 4 has no built-in ARIA role locator. We write a helper that builds XPath expressions combining two strategies:

  • Semantic HTML — map roles to HTML tags. A "button" is a <button>.
  • Explicit ARIA — catch <div role="button"> components.
  • XPath union operator | — OR: either match works.
tools.py — _by_role_and_text()
ROLE_TO_ELEMENTS = {
    "button":   "button",
    "link":     "a",
    "heading":  "h1|h2|h3|h4|h5|h6",
    "checkbox": "input[@type='checkbox']",
    "listitem": "li",
}

def _by_role_and_text(role, text):
    tag = ROLE_TO_ELEMENTS.get(role, "*")
    return (
        f"//{tag}[contains(normalize-space(),'{text}')]"
        f" | //*[@role='{role}']"
          f"[contains(normalize-space(),'{text}')]"
    )
XPath union semantics

//button[...] catches semantic HTML  ·  | //*[@role='button'][...] catches explicit ARIA attributes. Together they cover both patterns in every browser.

from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
from selenium.webdriver.common.by import By

if name == "click":
    xpath = _by_role_and_text(
        inputs["role"], inputs["text"]
    )
    el = WebDriverWait(driver, 5).until(
        EC.element_to_be_clickable((By.XPATH, xpath))
    )
    el.click()
    return f"Clicked {inputs['role']} '{inputs['text']}'"
fill implementation
if name == "fill":
    label_xpath = f"//label[contains(
        normalize-space(),'{inputs['label']}')]"
    try:
        label = driver.find_element(By.XPATH, label_xpath)
        input_id = label.get_attribute("for")
        el = driver.find_element(By.ID, input_id)
    except Exception:
        el = driver.find_element(By.XPATH,
            f"//input[@placeholder='{inputs['label']}']")
    el.clear()        # always clear first
    el.send_keys(inputs["value"])
BiDi Extras

Three tools the
other drivers can't offer.

Tool 1
get_browser_logs
Retrieves last 10 entries from the JS console. Level: INFO, WARNING, SEVERE. System prompt instructs: call after every navigation.
Tool 2
execute_script
Run arbitrary JavaScript, get the return value as a string. Fills the gap when accessible elements don't exist: localStorage, canvas, computed CSS.
Tool 3
intercept_network
Block requests matching a URL pattern via CDP. Test error states, offline behaviour, missing images. Chrome/Edge only — agent skips gracefully on Firefox.
tools.py — get_browser_logs + execute_script
if name == "get_browser_logs":
    logs = driver.get_log("browser")
    if not logs: return "No browser logs."
    return "\n".join(f"[{e['level']}] {e.get('message')}" for e in logs[-10:])

if name == "execute_script":
    result = driver.execute_script(f"return {inputs['expression']}")
    return str(result)
agent.py · Browser Setup

Chrome and Firefox —
one function, one flag.

agent.py — _build_driver()
def _build_driver(browser: str, headless: bool):
    if browser == "chrome":
        options = webdriver.ChromeOptions()
        options.set_capability("goog:loggingPrefs", { "browser": "ALL" })
        # ↑ MUST be set before driver starts — cannot add later
        if headless: options.add_argument("--headless=new")
        return webdriver.Chrome(
            service=ChromeService(ChromeDriverManager().install()),
            options=options
        )

    if browser == "firefox":
        options = webdriver.FirefoxOptions()
        options.set_capability("webSocketUrl", True)    # BiDi transport
        options.set_preference("remote.active-protocols", 3) # Classic + BiDi
        if headless: options.add_argument("--headless")
        return webdriver.Firefox(
            service=FirefoxService(GeckoDriverManager().install()),
            options=options
        )
Chrome logging must be set at startup

goog:loggingPrefs cannot be added after the driver starts. Without it, get_browser_logs silently returns nothing.

TypeScript

Builder API,

tools.ts — Chrome
import { logging } from "selenium-webdriver";
import * as chrome from "selenium-webdriver/chrome";

const prefs = new logging.Preferences();
prefs.setLevel(logging.Type.BROWSER, logging.Level.ALL);

const options = new chrome.Options()
    .windowSize({ width: 1280, height: 800 })
    .setLoggingPrefs(prefs);

if (headless) options.addArguments("--headless=new");

return new Builder()
    .forBrowser("chrome")
    .setChromeOptions(options)
    .build();

    .forBrowser("firefox")

    .build();

Module 04 · Key Takeaways

Five things
to carry forward.

1
XPath union covers both semantic HTML and explicit ARIA
//button[...] catches <button> elements. | //*[@role='button'][...] catches component libraries that use div + ARIA. Together: everything.
2
goog:loggingPrefs must be set before the driver starts
Pass it in ChromeOptions before calling webdriver.Chrome(). Can't be added later. Without it, get_browser_logs silently returns nothing.
3
Cross-browser is one command-line flag
python run.py firefox. In CI, run both as parallel jobs. Use fail-fast: false in your matrix so both finish reporting even if one fails.
4
execute_script fills the gap for non-accessible state
localStorage, canvas, computed properties, event tracking — anything JavaScript can read, the agent can verify.
5
intercept_network is Chrome/CDP only
On Firefox the agent receives a FAIL and should skip gracefully. Add "skip network interception on Firefox" to the system prompt.
eval checkpoint

Eval Your Selenium BiDi Agent

BiDi gives you two eval angles that Playwright can't match:

Cross-browser consistency

Run the same golden scenario on Chrome and Firefox. Compare findings with deterministic judge. A delta > 20% keyword coverage between browsers = a real portability bug.

Log capture coverage

For scenarios that trigger JS errors, verify get_browser_logs appears in the tool call sequence and the finding mentions the console error.

Cross-browser consistency eval is free — you already have two drivers. Add it to every scenario that touches form submission or dynamic content.
