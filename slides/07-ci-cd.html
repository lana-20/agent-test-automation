The Two Differences

Two things change
in CI. Both already solved.

LocalCI (GitHub Actions)Already handled?
BrowserHeaded (visible window)Headless (no display)--headless flag on every module
API key.env fileGitHub Secret → env varSame env var name: ANTHROPIC_API_KEY

The code doesn't change between local and CI. Only the execution environment changes — and both differences are handled by configuration, not code.

agents.yml · Trigger and Jobs

Two gates.
Before and after merge.

agents.yml — trigger
on:
  push:
    branches: [main]
    # Runs after every merge to main
  pull_request:
    branches: [main]
    # Runs before a PR can merge
agents.yml — jobs
jobs:
  python:
    name: Python (Module 06)
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: 06-full-framework/python
    # ↑ No cd needed in every step

  typescript:
    name: TypeScript (Module 06)
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: 06-full-framework/typescript

Two jobs run in parallel. One for Python, one for TypeScript. Both report to the same PR check.

agents.yml · Install Steps

--with-deps is not
optional on Ubuntu.

agents.yml — install steps
- uses: actions/setup-python@v5
  with:
    python-version: "3.11"
    cache: pip
    cache-dependency-path: 06-full-framework/python/requirements.txt
    # ↑ Dependencies cached by requirements.txt hash → saves 20-30s

- name: Install dependencies
  run: pip install -r requirements.txt

- name: Install Chromium
  run: playwright install chromium --with-deps
    # ↑  --with-deps installs system libraries Chromium needs on Ubuntu
  #     WITHOUT it: browser starts, then crashes on missing shared objects
  #     This is the most common CI setup mistake

- name: Run test suite
  run: python run.py --headless
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
Warning

playwright install chromium (without --with-deps) will silently fail on GitHub's Ubuntu runners. The browser starts and immediately crashes. Always use --with-deps.

agents.yml · Artifact Upload

if: always() —
you need the report most when it fails.

agents.yml
- name: Write step summary
  if: always()
  run: |
    python step-summary.py report.json \
      >> $GITHUB_STEP_SUMMARY

- name: Upload reports
  uses: actions/upload-artifact@v4
  if: always()
  with:
    name: report-python
    path: |
      06-full-framework/python/report.md
      06-full-framework/python/report.json
Default behavior (without if: always())
If the test run step fails, GitHub skips all subsequent steps. No artifact. No summary. Nothing to diagnose the failure with.
With if: always()
Upload and summary steps run regardless of whether the suite passed or failed. Artifact available exactly when you need it.
$GITHUB_STEP_SUMMARY

step-summary.py
with open(path) as f:
    report = json.load(f)

summary = report["summary"]
icon = "✅" if summary["failed"] == 0 else "❌"

lines = [
    f"## {icon} Test Results",
    "| Total | Passed | Failed | Bugs |",
    f"| {summary['total']} | ...",
    "### Scenarios",
    "| ID | Title | Result |",
]
for r in report["results"]:
    status = "✅ PASS" if r["passed"] else "❌ FAIL"
    lines.append(f"| {r['scenarioId']} | {r['title']} | {status} |")

print("\n".join(lines))

agents-smoke.yml · Matrix Strategy

All four implementations,
one job definition.

agents-smoke.yml
on: workflow_dispatch  # manual only

jobs:
  test:
    strategy:
      fail-fast: false  # all jobs finish
      matrix:
        include:
          - name: Playwright (Python)
            dir: 02-single-agent-playwright/python
            run: python run.py --headless

          - name: Playwright (TypeScript)
            dir: 02-single-agent-playwright/typescript
            run: npm run start:headless

          - name: Vibium (Python)
            dir: 03-single-agent-vibium/python
            run: python run.py --headless

          - name: Selenium BiDi Chrome (Python)
            dir: 04-single-agent-selenium-bidi/python
            run: python run.py chrome
fail-fast: false

Default behaviour: if one matrix job fails, GitHub cancels the rest. You get a partial picture. fail-fast: false lets all jobs run to completion — full cross-implementation report.

workflow_dispatch only

The smoke suite runs all 4 implementations. Run it manually after dependency updates, not on every push — it consumes more API tokens.

Cost Management

Run only when
code actually changes.

agents.yml — paths filter
on:
  push:
    branches: [main]
    paths:
      - "0[2-6]-**"     # modules 02-06
      - ".github/**"    # workflow files
      - "07-ci-cd/**"   # scripts
  pull_request:
    branches: [main]
    paths:
      - "0[2-6]-**"
      - ".github/**"

# README change → no run
# Transcript change → no run
# requirements.txt change → runs

Module 07 · Key Takeaways

Six things
to carry forward.

1
--headless and ANTHROPIC_API_KEY are the only differences
The code is identical between local and CI. Configuration handles both.
2
--with-deps on Ubuntu — always
Without it the browser crashes on missing system libraries. Not optional.
3
if: always() on artifact upload and summary
You need the report most when the run fails. Default skipping behaviour is exactly wrong.
4
fail-fast: false for cross-implementation matrix
Get the full picture — all implementations finish even if one fails.
5
$GITHUB_STEP_SUMMARY renders Markdown in the PR
No one wants to dig through logs to find a verdict. Put the table in the summary.
6
paths filter keeps CI costs predictable
Only run the suite when code in the module directories actually changes.
